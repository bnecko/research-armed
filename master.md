# research-armed

An adversarial anti-marketing research council. Instead of reproducing the first vendor-friendly,
SEO-optimized answer, it runs six researchers with opposed mindsets, makes them argue, grades every
source by who benefits from it, and returns one calibrated verdict — marketing, or the real thing —
without flattering you and without reflexively denying everything.

Three builds of one protocol, for three places research happens.

## The builds

| Build | For | Entry point |
|---|---|---|
| **Web** | Claude.ai / ChatGPT — drag-and-drop, one context, no real subagents | `dist/research-armed.md` |
| **Codex** | Codex CLI / app — real subagents when available, sequential fallback | `codex/skills/research-armed/` |
| **Claude Code** | Claude Code — real parallel subagents + optional deterministic Workflow | `claude-code/.claude/` |

## Install and use

**One command (Claude Code + Codex):**
```sh
./install.sh
```
It builds the artifacts and installs both skills into `~/.claude` and `~/.codex`. After that you just
ask — `/research-armed` in Claude Code, `$research-armed` in Codex, or plain English ("is X marketing or
real?"). The skill does the scoping, searching, debate, and verdict; you supply only the subject. The
manual steps below are the same thing by hand.

**Web.** No install. In a fresh chat (turn on Research / Deep Research / web search if you have it),
drag in `dist/research-armed.md` plus the material to evaluate, and say "run this on the attached." To
share, send the one file or paste its contents.

**Codex.** Copy the skill into your Codex skills dir, then invoke it by name or let its description
trigger it:
```sh
cp -R research-armed/codex/skills/research-armed ~/.codex/skills/
```
Use `ln -s` instead of `cp -R` if you want it to track repo updates.

**Claude Code.** Copy the skill and the five council agents into your config, then run `research-armed`
(or describe the task and let the description trigger it):
```sh
cp -R research-armed/claude-code/.claude/skills/research-armed ~/.claude/skills/
cp research-armed/claude-code/.claude/agents/ra-*.md           ~/.claude/agents/
```
For a deterministic, schema-validated run, use the optional Workflow at
`claude-code/workflow/research-armed.workflow.js` — pass the claim or material as `args`, and
`{mode:"deep"}` for the broader pass. It is the highest-fidelity option and is not part of the default
skill install.

A note on Codex `AGENTS.md`: do not paste the council into your global `AGENTS.md`. It is an on-demand
skill, and `AGENTS.md` is an always-on operating contract with a 32KiB cap. At most add one line
pointing at the skill.

## How it is built (read before editing)

The protocol is written once in `core/` and `shells/`; `build.sh` fans it out into all three builds so
they cannot drift. Run it after any edit:
```sh
./build.sh
```

**Edit only these:**
- `core/*.md` — the protocol (personas, source grading, debate round, verdict scale, output contract).
  The single source of truth.
- `shells/*.md` — the three thin platform adapters (`web`, `codex`, `claude-code`).
- `claude-code/.claude/agents/ra-*.md` — the five subagent bindings (persona content stays in `core/`).
- `claude-code/workflow/research-armed.workflow.js` — the optional Workflow. It inlines a condensed
  copy of the protocol (its agents run in fresh contexts and cannot read `reference/`), so re-sync it by
  hand when you change `core/`.
- `codex/skills/research-armed/agents/openai.yaml` — Codex UI metadata.
- `build.sh`, `master.md`.

**Never hand-edit (generated by `build.sh`):** `dist/research-armed.md`, both `*/SKILL.md`, and both
`*/reference/`. A change made there is lost on the next build.

## The protocol in one paragraph

A pre-flight **Scoper** de-loads your question into a neutral, falsifiable claim. Four vectors then work
in parallel: a **Prosecutor** and a **Defender** with symmetric burdens (popularity is not evidence
either way), a **Field Investigator** that leaves the SEO'd surface for forums, complaint boards, and
teardowns, and a neutral **Source Grader** that scores every source by independence and runs the
marketing-machinery scan. They cross-examine — each must concede a point and rebut the strongest
opposing one — and a separated **Adjudicator** assigns one of five verdict levels (from pure marketing
to genuinely substantiated, with a real "real core wrapped in marketing" middle), anchored to the
evidence ledger and guarded against both flattery and reflexive denial.
